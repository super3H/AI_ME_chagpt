# Skill 11: 专利交底书草案生成算子

## 目标
将 `skill_10` 的最高分方向转为结构化专利交底书草案，供研发与代理人快速迭代。

## 参考文献（默认加载）
- `30_skills/patent/references/patent_reference_profile.md`
- `30_skills/patent/references/patent_style_guide.md`
- `30_skills/patent/references/1-附件一-发明实用新型技术交底书--广域容量巡检.docx`
- `30_skills/patent/references/1-附件一-发明实用新型技术交底书--数据流动.docx`
- `30_skills/patent/references/1-附件一-发明实用新型技术交底书--数据流动调度策略.docx`

## 输入
- 来自 `skill_10` 的入选方向
- 目标申请地区
- 可公开与不可公开边界
- 可提供的实验/数据/流程图

## 草案骨架
1. 第一部分：技术方案说明
2. 技术背景与最接近现有方案
3. 现有技术缺点与发明目的（一一对应）
4. 技术方案详细阐述（分阶段、分步骤）
5. 关键点与保护点
6. 替代方案与落地可能性
7. 第二部分：检索报告（专利库/搜索引擎/论文）
8. 权利要求草案（独立权利要求 + 从属权利要求）

## 执行步骤
1. 先写“一句话发明定义”。
2. 抽象成“必要技术特征集合”（用于独立权利要求）。
3. 补充可替换特征（用于从属权利要求扩展保护范围）。
4. 写 2-3 个实施例，覆盖不同参数/场景。
5. 输出风险提示：公开过度、保护过窄、规避路径。

## 输出格式
```md
发明名称：
一句话发明定义：
目标地区：

【第一部分 技术方案说明】
【技术领域】
...

【背景技术】
...

【发明内容】
...

【有益效果】
...

【具体实施方式】
实施例1：
实施例2：

【权利要求草案】
独立权利要求1：
从属权利要求2：
从属权利要求3：

【申请风险与复核清单】
- 公开充分性风险：
- 保护范围风险：
- 规避风险：
- 需代理人复核事项：

【第二部分 检索报告】
- 检索关键词：
- 中国联通专利数据库检索结果：
- 最相关专利文献分析：
- 公共搜索引擎检索结果：
- 与最接近现有技术对比分析：
- 期刊/学术论文检索：
```

## 强制约束
- 不输出“已授权概率”这类确定性承诺。
- 任何法律结论都要注明“代理人/律师复核”。
- 草案叙事与分段风格需对齐三份参考交底书，禁止改成通用营销文风。
