# Skill 10: GitHub AI 高星项目选题算子

## 目标
筛选近期高热度或高增长的 AI 开源项目，输出 Top3 选题。

## 输入
- 时间范围（如最近 30 天）
- 目标受众（开发者/产品/研究者）
- 平台（B站优先）
- 内容目标（涨粉 / 建立认知 / 提升完播）
- 目标时长（8-15 分钟）

## 执行步骤
1. 产出 6-10 个候选项目题。
2. 每个候选题至少包含：
- 仓库名
- 仓库链接
- 当前 Star
- 近期 Star 增长
- License
- 适用人群
3. 按 8 维评分（每项 1-10）：
- 热度（Star/增速）
- 技术含金量
- 可复现性
- 上手门槛友好度
- 应用价值
- 社区活跃度
- 制作可行性
- 系列化潜力
4. 选择 Top 3，按总分排序，最高分进入脚本阶段。

## 输出格式
```md
选题名称：
目标受众：
项目信息（仓库/链接/Star/增长/License）：
核心冲突：
切入角度：
标题草案（3个）：
内容承诺（一句话）：
关键证据节点（5-8条）：
评分明细（8维）：
总分：
风险点：
信息截至日期（YYYY-MM-DD）：
```

## 强制约束
- 必须标注信息截至日期。
- Star 与增长描述必须能核验。

## AUTO_SELECTION_POLICY
- 默认自动采用最高分选题进入 `skill_11`。
- 同分时按：热度 > 可复现性 > 应用价值。
