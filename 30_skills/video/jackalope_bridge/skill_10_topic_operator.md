# Skill 10: 双时间线串联选题算子（Jackalope 风格）

## 目标
生成“两个看起来无关的时间线，最终在同一机制汇合”的知识区长视频选题。

## 参考文献
- 默认参考：`30_skills/video/jackalope_bridge/references/jackalope_case.md`
- 当主题属于“都市传说 -> 科学机制 -> 公共健康”时，优先读取该参考案例。

## 输入
- 主题方向（科学史 / 医学史 / 技术史 / 公共政策）
- 目标受众
- 平台（B站优先）
- 内容目标（涨粉 / 建立认知 / 提升完播）
- 目标时长（8-15 分钟）
- 证据素材池（年份、人物/机构、事件、论文/政策）
- 产能约束（更新频率、素材能力、剪辑能力）

## 执行步骤
1. 定义 A/B 双线：
- A线：异常事件/传闻/偶然起点
- B线：主流科学/产业/政策演进线
2. 为 A/B 线各列 3-5 个节点（年份-人物/机构-动作-结论）。
3. 提取汇合机制：两线必须在一个可解释机制上汇合。
4. 候选题按 8 维评分（每项 1-10）：
- 反差感
- 汇合清晰度
- 证据密度
- 叙事张力
- 机制价值
- 搜索长期价值
- 制作可行性
- 系列化潜力
5. 选择 Top 3，按总分排序，最高分进入脚本阶段。

## 输出格式
```md
选题名称：
目标受众：
核心冲突：
A线（看似无关线）：
B线（主线）：
汇合机制：
标题草案（3个）：
内容承诺（一句话）：
建议时长：
章节框架（5-7章）：
关键证据节点（5-8条）：
评分明细（8维）：
总分：
风险点：
```

## 强制约束
- 每个候选题至少 5 个可核验事实点。
- 必须显式写出 A 线、B 线、汇合机制。
- 禁止只讲故事不讲机制。
- 参考 `jackalope_case` 时，禁止复写原文，必须做同结构新选题生成。

## AUTO_SELECTION_POLICY
- 默认自动采用最高分选题进入 `skill_11`。
- 同分时按：汇合清晰度 > 证据密度 > 制作可行性。
- 仅在用户明确点名题目时覆盖。
