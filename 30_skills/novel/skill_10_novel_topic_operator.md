# Skill 10: 小说选题与大纲筛选算子

## 目标
从用户输入中产出 Top3 小说方向，完成结构化评分，自动选择最高分方向进入正文创作阶段。

## 参考能力
- 若题材为“副本/乐园/无限流/规则怪谈/考试闯关”，优先调用：
  - `30_skills/novel/skill_12_instance_fiction_blueprint_operator.md`

## 输入
- 题材方向（科幻/悬疑/言情/历史/奇幻/现实等）
- 目标读者（年龄层、偏好）
- 篇幅目标（短篇/中篇/长篇/连载）
- 风格偏好（叙事节奏、文风、视角）
- 禁忌项（题材禁区、表达边界）

## 执行步骤
1. 生成 3 个候选方向，每个方向给出：
- 核心命题
- 主角设定
- 核心冲突
- 世界观边界
- 结局走向
2. 对每个方向做 8 维评分（1-10）：
- Hook 强度
- 冲突可持续性
- 人物成长空间
- 情节推进弹性
- 题材差异化
- 连载可扩展性
- 商业化潜力
- 写作可执行性
 - 沉浸场景可写性（加权项）：是否能落到“可走的空间 + 可触摸的物件 + 可见的机制/代价”；纯概念/纯流程会显得无聊，需扣分
 - 恐怖素材可用性（加权项）：恐怖点是否来自“案件故事与人性压迫”，能否转成具体场景与事件链；纯概念恐怖扣分
3. 输出 Top3 排序，自动选择最高分进入 `skill_11_novel_draft_operator`。

## 输出格式
```md
方向名称：
核心命题：
主角设定：
核心冲突：
世界观边界：
结局走向：
章节大纲（5-8章）：
评分明细（8维）：
总分：
主要风险：
```

## 强制约束
- 不写空洞设定，必须给可写的冲突与事件链。
- 避免设定和题材强冲突（如硬科幻中出现无解释玄学关键解法）。
- 明确“题材承诺”，避免中后期风格漂移。

## AUTO_SELECTION_POLICY
- 默认自动采用总分最高方向进入 `skill_11`。
- 同分时按：冲突可持续性 > 人物成长空间 > 写作可执行性。
