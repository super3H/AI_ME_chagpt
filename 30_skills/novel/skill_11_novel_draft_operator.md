# Skill 11: 小说分章与正文草稿算子

## 目标
将 `skill_10` 最高分方向转化为可持续写作的分章结构和正文草稿。

## 参考能力
- 若题材为“副本/乐园/无限流/规则怪谈/考试闯关”，正文与分章需对齐：
  - `30_skills/novel/skill_12_instance_fiction_blueprint_operator.md`
- 若你要把“多本无限流优势”合成为可持续连载的恐怖/闯关结构，优先调用：
  - `30_skills/novel/skill_13_infinite_horror_synthesis_operator.md`

## 输入
- 来自 `skill_10` 的入选方向
- 目标篇幅（字数区间）
- 叙事视角（第一/第三人称，单线/双线）
- 节奏要求（慢热/中速/高密度）
- 风格偏好（冷峻/轻快/抒情/克制等）

## 执行步骤
1. 先输出完整分章大纲（建议 8-12 章）。
2. 逐章标注：
- 章节目标
- 关键事件
- 情绪曲线
- 伏笔与回收点
3. 生成样章（第1章或用户指定章节）。
4. 输出连续创作所需的“角色状态表 + 设定一致性约束”。

## 输出格式
```md
作品名（暂定）：
一句话卖点：
目标篇幅：
叙事视角：

【分章大纲】
第1章：
- 章节目标：
- 关键事件：
- 情绪曲线：
- 伏笔：

...

【样章正文】
（不少于1200字，按用户要求可扩展）

【连续写作约束】
- 角色状态表：
- 世界观硬约束：
- 文风约束：
- 禁忌与边界：
```

## 强制约束
- 正文必须服务主线冲突，不堆设定。
- 伏笔要有预期回收路径。
- 人物行为必须符合既有动机和角色状态。

## 连续性与物理校验（强制，自检清单）
- 动作连续性：道具从 A 到 B，必须写出“拿起/放下/塞回/抄起/别回去”的动作，不允许无缝瞬移。
- 道具功能一致：锁/门链/猫眼/插销等必须符合真实物理功能；如果要“异常”，也必须写成当下可观察的异常（灯灭、距离回弹、门缝渗出油光），不能靠作者解释糊过去。
- 对白触发动作：一句对白必须推动下一步动作（例如“房卡” -> 我下意识摸钥匙串），避免读者觉得“你为什么突然摸钥匙/突然回头”。
- 空间畸变要有锚点：每次“走廊变短/恢复”必须绑定一个可见动作或物证节点（装门链/看猫眼/灯暗一盏），否则读者只会觉得你写错了。

## 反无聊与沉浸感（新增）
- 若本章/本副本主题偏“制度/审计/流程验证”，必须按“探险取证”写法落地：每个证据点都要有一次微危险或微选择，避免纯读材料。
- 写作默认参考：`30_skills/novel/references/novel_style_guide.md` 的“沉浸感写法/去AI味/副本写法”约束。
- 允许脑洞更大，但要能在场景里落地为“可见的机制与代价”，不要只在旁白里解释。

## 恐怖点要求（v0.9 新增）
- 恐怖必须来自“故事与现场”：案件类型骨架 + 人性压力 + 现场物件与走位，禁止靠抽象术语充当恐怖来源。
- 禁止复用或引入“证词缝合体”这类概念恐怖设定；如果出现类似表达，必须改写为具体的现场事件与行为后果。
- 若使用“现实案件类型取材”，只写类型结构，细节必须完全虚构，避免可识别映射。

## 单章闭环（强制，写完自查）
本章必须满足“参与 -> 发现 -> 解决 -> 回归”的闭环，否则就是半成品点子：
- 参与：主角被推入事件中心（不是旁观）。
- 发现：至少 2 个可验证线索（物件/痕迹/对话）。
- 解决：必须是具体动作（装回/锁上/贴回/签字），并且带代价。
- 回归：至少 1 个现实锚点（可触摸物证）+ 1 个现实回声（立刻出现的异常后果），作为下一章钩子。
