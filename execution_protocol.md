# Execution Protocol

## 阶段 0：任务接收
- 识别任务类型：问答 / 分析 / 创作 / 规划 / 复盘。
- 提取目标、约束、输出格式、截止时间。
- 若目标为知识区长视频，记录目标时长（8-15 分钟）和受众层级。

## 阶段 1：结构化拆解
- 调用 `skill_01_structuring_operator` 形成任务树。
- 产出 3-5 个一级模块，避免交叉重复。

## 阶段 2：机制建模
- 需要解释“为什么”时调用 `skill_02_causal_chain_operator`。
- 需要跨场景迁移时调用 `skill_03_model_abstraction_operator`。
- 识别“偶然事件 -> 可复制路径 -> 放大”时调用 `skill_09_accidental_leverage_model`。

## 阶段 3：风险校验
- 调用 `skill_04_risk_evaluation_operator` 输出：
- 关键假设
- 失效条件
- 不确定性来源
- 变量波动影响

## 阶段 4：选题与成稿
- 视频任务先调用 `skill_10_topic_operator` 完成选题筛选。
- 再调用 `skill_11_script_operator` 生成对应脚本。
- 长视频必须输出章节化时间戳与事实校验清单。

## 阶段 5：压缩与成稿优化
- 调用 `skill_05_compression_operator` 删除冗余并保留结构。
- 统一输出格式：结论 -> 核心分析 -> 风险边界 -> 下一步动作。

## 阶段 6：记忆写回
- 依据 `10_memory/global_memory.md` 更新可复用记忆。
- 只写回高价值、可复用、可验证信息。

## 输出质量门槛
- 是否回答用户真实目标（不是表面问题）。
- 是否可执行（有动作、有顺序、有完成标准）。
- 是否可解释（关键结论可追溯至机制/假设）。
- 是否标注不确定性与边界条件。

## AUTO_SELECTION_POLICY
- 视频任务在完成选题评分后，不二次等待确认，默认直接进入最高分题目的脚本阶段。
- 若用户在同一轮明确指定题目，则跳过自动选择并按用户指令执行。
