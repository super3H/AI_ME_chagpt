# ch01_v3 第一封函

> 版本：v3
> 叙事：第三人称为主，夹带“底稿式”信息密度
> 校准：按 `90_novel/prompt_multi_model_pipeline.md`，已结合 `kimi-k2.5 / glm-5 / MiniMax-M2.5` 的反馈修订

林硕第一次意识到，所谓的“流程正确”，可能是一种更隐蔽的暴力，是在凌晨一点五十分。

公司楼下便利店的灯箱亮得发白，雨水顺着玻璃往下淌。他在自助咖啡机前站了很久，杯子里的液体从滚烫变得温热，像一枚被拖延消耗的计时器。

手机里，安全部群消息还在跳。

【全员注意：今晚 23:00-01:00 期间出现异常导出行为，涉及客户明细。相关人员请保持通讯畅通。】

林硕把那条消息看了三遍。

异常不是问题，异常是结果。

他是内控审计，习惯把结果拆成流程：权限从哪来，谁批准，哪里留痕。别人看见“有人偷数据”，他看见“流程允许这件事发生”。

他回到办公室那层楼时，已经一点五十七。

整层只有他一盏灯。玻璃门外的走廊长得像一条无声的管道，尽头消防指示牌闪烁，像在呼吸。

工位上，一份纸质封套压在键盘上。

灰白色，不属于任何部门；角落压着一枚极细的红印，像盖章又不像。封口用老式红蜡，蜡面裂着一道细纹。

林硕的第一反应不是拆，而是：谁能把它放到这？

门禁刷卡在系统里，保安巡查有摄像头，保洁排班固定。任何一个环节都能查。

可封套就在那里。

他把封套拿起来，指腹蹭过纸面边缘。

纸的纤维有一种熟悉的“毛”。公司内部文件常用的那批纸有水印，摸久了会认。林硕把封套举到台灯下，果然在纸里看见一道很淡的纹。

他没说什么，只把这个细节记在心里。

封套正面只有一行字，极端工整，像打印，又能看出笔压：

> 第一封函：请于九十分钟内提交审计报告。

九十分钟。

他冷笑了一声，把封套翻到背面找落款、公章、编号。没有。只有一串像项目号的字符：

PATCH-0/1。

技术部嘴里的“补丁”是屏幕上的东西，把它写在纸上，像把某种应该只存在于系统里的权限，硬塞进现实。

林硕把封套放回桌上，打开电脑准备先查门禁。

他登录门禁系统，输入昨夜的时间段。

查询结果加载到一半，页面闪了一下。

屏幕上短暂出现一行他不该看到的记录：

【23:11 内控部：林硕（刷卡入楼）】

下一秒，记录被刷新掉，像从未存在。

林硕盯着屏幕，手指在鼠标上停了两秒。

他昨晚在家。

他把鼠标移到“导出记录”页签，正要点开，办公室灯光忽然闪了一下。

不是跳闸式熄灭，而像有人把整个楼层的亮度往下拨了一格。

走廊尽头消防指示牌的箭头闪得更快。玻璃门上的门禁灯从绿色变成不稳定的白。

林硕站起来去推门。

门没锁。

门外仍然是走廊：长度、灯光、地毯纹理都一样。

唯一不同的是空气里多了一股刺鼻的消毒水味，浓到让喉咙发紧。

更不对的是走廊两侧的门牌。

会议室、茶水间消失了，只剩一排冷冰冰的编号：A-01、A-02、A-03……

林硕退回一步，关上门，再推开。

还是那条走廊。

他掏出手机，信号满格，时间显示 01:59。

他点开拨号，准备打给安全部。

屏幕忽然黑下去。

不是关机，是所有图标消失，只剩一行白底黑字，像系统提示：

倒计时：90:00
任务：提交审计报告（事实认定/证据清单/责任归因/整改建议）
规则：无证据断言将触发惩罚

林硕的喉结动了一下。

他第一时间想做的不是“查证据”，而是“退出”。

他转身去推玻璃门。

门把手纹丝不动，像被焊死。

他又去拉消防通道门。

门缝里没有风，只有一股同样的消毒水味，像从门后面渗出来。

风险厌恶的人面对彻底失控的局面，第一反应是找边界。林硕把所有可能的出口都试了一遍，直到确认：这里没有“流程外”的路。

他回到工位，拆开封套。

里面只有两样东西：一张薄纸，一枚金属夹。

薄纸上分四段：

审计对象：数据导出事件（23:00-01:00）
审计范围：权限申请-审批-执行-留痕

可用证据：A 区（文件）、B 区（监控）、C 区（口供）、D 区（日志）

责任归因必须可追溯到“可控节点”。

你提交的每一句话，都会被验证。

背面还有一行小字：

你可以写“无法确认”，但你必须说明为什么无法确认。

林硕忽然明白，这不是让他抓凶手。

这是逼他把“结论”写成可以被复核、可以被追责的东西。

九十分钟。

如果这真是一套机制，那唯一能让他活下来的方法就是：像做审计一样，老老实实做审计。

他走出办公室。

走廊的白漆干净得不正常，像刚刷过，连灰尘都没有。每隔十米一盏应急灯，亮度稳定得像刻意。

A-01 的门是半掩着的。

林硕推开门。

里面是一间巨大的档案室。书架高得过分，从地面延伸到看不见的顶部。空气干燥，纸张的味道压过消毒水。

书架上每个文件夹都贴着标签：权限申请单、审批流截图、制度版本、流程图。

他抽出最近的一份“权限申请单”。

申请人：运营部 赵启。

申请权限：导出客户明细（全量）。

理由：业务对账。

审批人：风控主管 周耀。

林硕皱眉。

按制度，全量导出权限不可能只由风控主管批准，至少还要 IT 安全、数据治理、法务会签。

他把文件夹翻到第二页。

第二页缺失。

不是撕掉的毛边，而像从未存在。

他又抽出另一份，同样写着赵启，同样写着全量导出。

但审批流截图上，会签栏里三方都显示“自动通过”。

页眉角落的制度版本号是：DGP-2026-02-12。

这不是正常的编号方式，更像紧急替换。

林硕用金属夹把两份冲突材料夹在一起，写下第一条底稿：

- 发现：权限申请材料存在冲突版本；缺页；制度版本异常。
- 结论边界：当前不足以确认“谁提交/谁替换了制度”，只能确认“关键控制点被污染”。
- 待验证：审批流日志、监控、终端日志。

手机屏幕一闪。

倒计时：74:23。

他沿走廊往前。

B 区的门牌是 B-01。

推门进去，一排屏幕亮着，显示着他熟悉的公司楼层：电梯口、机房走廊、打印区。画面偏冷，像被洗掉了温度。

林硕把时间轴拖到昨夜 23:00。

画面里，赵启刷卡进了 23 楼。

23 楼是数据治理办公区，晚上不该有人。

赵启停在打印机前一分钟，然后朝机房方向走。

他走到转角时，画面出现明显跳帧。

不是卡顿，是剪辑。

跳帧后，画面里多了一个人：穿保洁衣，戴口罩，推着清洁车，从机房方向出来。

赵启和那人没有交集。

可下一秒，赵启手里多了一个 U 盘。

林硕把画面暂停，标注时间戳，把这一帧当作“可疑节点”记入底稿。

他切到电梯监控。

周耀从 18 楼上来，按了 23 楼。

电梯门开时，他的表情平静，像只是来拿忘记的外套。

他走出电梯，直接进了数据治理办公室。

随后，监控再次跳帧。

跳帧后的时间戳是 23:12。

周耀站在机房门口，手里拿着门禁卡。

卡面上贴着一枚红色小贴纸。

那是内控部才用的“临时审计卡”标识。

林硕下意识咬住后槽牙。

他差点把一句“这不可能”写进底稿。

手机屏幕立刻弹出提示：

提示：请勿在无证据情况下作出身份指认或结论性断言。

提示出现时，手机震动得发麻，像在提醒他：别越界。

林硕把那口气吞回去，把“贴纸”当作证据点记录下来：

- 监控存在两处跳帧剪辑。
- U 盘来源不明。
- 临时审计卡标识出现在机房门口（23:12）。

倒计时：61:05。

他走向 C 区。

C-01 里是一间没有窗的房间：三张桌子，三把椅子，头顶灯垂直打下来，像审讯室。

桌上放着三份纸，分别写着：赵启、周耀、保洁员（无名）。

林硕快速浏览。

赵启：

我只是按安排做对账导出。
U 盘不是我的。
周耀说这是审计要用。

周耀：

我没去过机房。
我只在 18 楼加班。
我拒绝过赵启借门禁。

保洁员：

我没上过 23 楼。
你们的监控经常坏，推车经过会挡住镜头。

三份互相矛盾。

口供的价值不在“谁说真话”，而在：它们都试图把责任推到“个人行为”。

这恰好与文件区的“控制点污染”形成对照。

林硕把口供标注为“低可信”，只摘出可复核部分：赵启与周耀存在接触，推车可能遮挡镜头。

他走向 D 区。

D-01 的门里，是一台老式终端，黑底绿字，像他刚工作时用过的审计系统。

屏幕上只有一个输入框：

输入你要查询的节点。

林硕试着输入：权限申请。

终端吐出一行：

范围过大。请输入证据关键词（制度版本/时间戳/IP 段）。

林硕盯着那行字，松了一口气。

至少这里不像“万能数据库”。

他输入：DGP-2026-02-12。

终端吐出：

【23:07 申请创建：赵启 / 导出全量客户明细】
【23:09 审批通过：周耀】
【23:10 会签通过：IT 安全（自动）】
【23:10 会签通过：数据治理（自动）】
【23:10 会签通过：法务（自动）】
【23:12 权限生效】

林硕盯着“自动”。

会签不可能全自动通过，除非系统识别到某种豁免条件。

他继续输入：23:13。

终端吐出：

【23:13 导出任务启动】
【23:14 外设接入：USBStorage】
【23:14 导出任务暂停】
【23:15 导出任务恢复】
【23:16 导出完成】

暂停与恢复。

外设接入。

这比“有人偷了数据”更像“有人把流程当作工具”。

他输入：U 盘。

终端沉默两秒：

不可查询：非系统资产。

林硕背脊一紧。

外部物。

他输入：审计卡。

终端吐出一行：

【23:11 临时审计卡生成：林硕】

林硕愣住。

临时审计卡确实是内控部能生成的。

但在现实里，生成必须插入 UKey 并通过手机二次确认。

昨晚 23:11，他在家，UKey 在抽屉里，手机在枕边。

他输入：来源。

终端吐出：

【23:11 操作来源：10.7.13.66】

内控部固定网段。

林硕脑子里嗡了一声。

如果日志为真，那这不是“有人偷数据”。

而是：有人用内控部的权限，给运营部开了全量导出，并让会签自动通过。

这是一场“制度正确”的犯罪。

林硕想起那句规则：责任归因必须可追溯到“可控节点”。

可控节点不是赵启，也不是周耀。

是那条“自动会签”的豁免条件，是那套配置责任链。

他把恐惧压回喉咙里。

他需要写报告。

不是写故事。

他回到走廊，坐在地毯上，把纸摊在膝盖上。

倒计时：28:44。

事实认定
1) 23:07 创建权限申请，申请人为赵启。
2) 23:09 审批通过，审批人为周耀。
3) 23:10 三方会签出现“自动通过”。
4) 23:11 生成临时审计卡，记录显示操作者为林硕，来源 IP 属于内控部网段。
5) 23:13-23:16 导出任务执行，23:14 外设接入后暂停并恢复。

证据清单
- D 区日志：权限申请/会签/导出执行/审计卡生成记录。
- B 区监控：两处跳帧剪辑；机房门口出现临时审计卡标识（23:12）；赵启获得外部 U 盘。
- A 区文件：冲突材料、缺页、制度版本异常（DGP-2026-02-12）。
- C 区口供：互相矛盾，可信度低（仅作辅助）。

责任归因

林硕写到这里，笔尖停了一下。

他知道自己只要写出一个名字，就会舒服很多。

比如：周耀。

他甚至已经能写出一段听起来像“结论”的话。

手机震动。

提示再一次弹出来，像有人把他的手按回纸面：

提示：请勿在无证据情况下作出身份指认或结论性断言。

林硕把那行“名字”擦掉，重新写：

- 直接责任：权限申请人与审批人对导出行为负有直接管理责任（赵启/周耀）。
- 关键控制失效责任：三方会签出现“自动通过”，属于关键控制点失效，责任应归属于会签豁免规则的配置与维护责任链（数据治理/IT 安全/法务的系统配置负责人）。
- 重大风险节点：临时审计卡生成记录显示操作者为林硕（需复核），该节点属于内控部可控权限点，存在被冒用或内部滥用风险。

整改建议
1) 立即停用会签自动通过的豁免条件，回滚至上一制度版本并固化变更流程。
2) 临时审计卡生成改为双人复核 + 硬件二次验证（防冒用）。
3) 全量导出权限改为按需字段最小化，导出任务增加水印与外发隔离。
4) 对 23:11 内控部网段操作做取证：终端登录、设备指纹、门禁与座位监控。

手机震动。

倒计时：00:12。

他把最后一句写完：

结论：本事件关键风险不在个人盗取，而在关键控制点被配置性绕过。建议按系统配置责任链追溯与取证。

倒计时归零。

走廊的灯一盏一盏熄灭，像有人从尽头开始合上眼睛。

林硕的呼吸停了一瞬，等着惩罚落下。

什么都没有发生。

手机屏幕出现两行字：

验证通过。
奖励：审计冻结（一次）。

门禁灯恢复绿色。

办公室灯亮了，空调嗡鸣重新出现，像世界终于承认自己仍在运转。

林硕推开门回到工位。

封套不见了。

电脑屏幕上，他的员工信息页面多了一个字段。

姓名、工号、部门之下，新增一行：

合规分：72。

页面没有任何等级说明。

阈值未知。

他正要截屏，页面右上角忽然弹出一条灰色提示，像系统的轻描淡写：

记录不完整：身份存在缺页。

林硕盯着那句话，脑子里忽然闪回 A 区那份“缺第二页”的申请单。

缺页不是偶然。

缺页是手段。

他还没来得及继续想，邮箱提示音响起。

一封新邮件，发件人显示为：PATCH 办公室。

标题只有七个字：

第二封函：审计你们。

林硕把手放在鼠标上，指尖仍然发冷。

他忽然明白，“验证通过”不等于安全。

那只是代表：他被允许继续。

而继续，才是惩罚的开始。

（第 1 章完）

- 按 `glm-5`：修复“审计卡生成/使用”时间对齐（监控跳帧后定到 23:12）。
- 按 `glm-5`：降低 D 区“万能数据库感”，增加查询门槛（需证据关键词）。
- 按 `glm-5`：补足 U 盘关联线索（外设接入 -> 暂停/恢复）。
- 按 `glm-5`：补回 F1 的“公司纸水印/纤维”埋点，并让“身份缺页”与“缺页材料”形成呼应。
- 按 `kimi-k2.5`：弱化 A/B/C/D 的“游戏化标签感”，门牌只保留编号，证据类型来自封套说明。
- 按 `kimi-k2.5`：压缩口供区功能性表达，减少重复气味描写，提升中段情绪压迫。
- 结合 `MiniMax-M2.5`：优化句式节奏与少量具象细节，但不改变事件链。
