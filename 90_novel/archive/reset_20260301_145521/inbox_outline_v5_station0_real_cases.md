# Outline v5（Station 0：设定加强版 / ch01写法更“副本引擎”）

## 第 1 章：终点站不认人

目标：主角第一次进入终点站，知道“出站证明”只是门槛；真正可怕的是“页码缺页”会回写到现实。

现实案件类型来源：地铁火灾/踩踏事故（彻底虚构化）

副本皮肤：废弃地铁站台（焦痕、泡沫水渍、撕裂广告牌、散落鞋印与票据）

关键冲突：
- 倒计时 60 分钟，闸机提示“身份验证中”
- 证据污染四件套：同号票/监控纯黑10秒/日志缺页/口供模板化
- 规则触发点明确：写入结论才惩罚
- 页码机制首次露面：站内出现“身份档案缺页”的物理提示；回到现实后变成具体提示

证据推进（四件套）：
1) 同号票据冲突：0417-0092（已出站01:41 vs 未进站）
2) 监控：01:41:12-01:41:22 纯黑；黑屏前出现疑似烟雾/捂鼻动作（不构成结论）
3) 日志：01:40 后整页缺失，断口整齐；下一页“清场失败/出口不认人”
4) 口供：两人模板句一致；逼出非模板锚点（甜味/糖水摊灯箱）并与监控末帧对上

行动线（必须走位清晰）：
闸机 → 站务玻璃房（日志/监控/终端）→ 站台柱后乘客（口供锚点）→ 回玻璃房写证明 → 回闸机提交 → 现实回波

规则触发（必须出现一次）：
主角在终端“责任归因”写下无证据结论（如“系统故障”），触发惩罚 + 弹窗“证据不足：结论不可写入”，迫使他改写成可控节点表述。

反转：
这不是“找凶手”，而是“找责任位”；关键节点被剪掉，系统逼你把事故写成可执行的归责材料。

章末钩子（更强）：
现实门禁：身份验证中；保安追问姓名，主角报“林硕”却被否认；
手机出现：合规分 + 身份缺页提示（可具体到页码标记，作为 v1→v2 的连载推进钩子）。
