## 第 1 章：终点站不认人

**目标**：主角首次被拉进0号线终点站，活着拼出"出站证明"。

**现实案件类型来源**：地铁火灾/踩踏事故（彻底虚构化）

**副本皮肤**：废弃地铁站台——灯光闪烁，广告牌撕裂，地上散落票据与鞋印。

**关键冲突**：
- 倒计时启动（60分钟），出口闸机紧闭
- 两份同编号票据内容冲突
- 监控录像关键10秒丢失
- 口供模板痕迹明显

**证据推进**：
1. 发现地上散落的票据，编号相同但内容不同（一张显示"已出站"，一张显示"未进站"）
2. 监控录像覆盖关键时段——10秒黑屏
3. 站务日志撕去一页，断口整齐
4. 两名"乘客"口供措辞高度一致，像复制粘贴

**反转**：
问题不是"谁偷了票/谁没刷卡"，而是"谁篡改了凭证链，制造了一个可归责的出站失败"。

**恐怖点**：
- 倒计时数字在墙上投影，每分钟变红一次
- 广播反复播放："请提交出站证明，否则将启动身份回收程序"
- 闸机口站着一个人形轮廓，没有脸

**行动线**：
- 主角被迫在站内走位，收集散落的证据
- 差点断言"肯定是系统故障"，被电流击穿手臂（无凭断言惩罚）
- 最终拼出"票据冲突源于后台篡改"的证明，闸机开启

**奖励/代价**：
- 奖励：首次出站权限
- 代价：现实系统中出现"合规分"字段，初始值100

**章末悬念**：
回到现实后，主角发现自己的门禁卡刷不开公司大门——系统显示"身份验证中"。

**台账更新**：F1（票据纸张纤维）、F2（合规分字段）

## 第 2 章：证据污染

**目标**：建立"证据权重与污染"机制，副本可重复恐怖。

**现实案件类型来源**：医院感染/医疗事故（彻底虚构化）

**副本皮肤**：医院住院部走廊——病房门牌号模糊，护士站无人，病历本散落。

**关键冲突**：
- 目击者口供高度一致（措辞模板化）
- 监控缺口覆盖关键一分钟
- 病历本撕去诊断页
- 护士站日志被水泡过，字迹模糊

**证据推进**：
1. 三名护士口供措辞完全相同："我当时在配药室，没有看到任何人进出"
2. 监控录像覆盖走廊尽头的病房门——关键一分钟黑屏
3. 某份病历的诊断页被撕，断口有撕痕
4. 配药室日志被水泡过，时间线断裂

**反转**：
污染不是为了让你选错，是为了让你"选得像你应该选的那样"——系统在引导你归责某个特定对象。

**恐怖点**：
- 走廊尽头站着穿护士服的人，没有脸
- 病房内传来呼吸机停止的声音
- 广播："请勿在无证据情况下归责医护人员"

**行动线**：
- 主角在走廊内取证，差点被"无凭断言"惩罚
- 发现口供模板与站务污染模板一致
- 选择归责"流程配置问题"而非个人，出站成功

**奖励/代价**：
- 奖励：冻结一次站务审查（被动触发）
- 代价：冻结触发时，无辜者被加速审查

**章末悬念**：
主角回到现实，发现公司内部调查启动——他被要求解释一次"异常门禁记录"。

**台账更新**：无新伏笔

## 第 3 章：现实回波

**目标**：展示奖励的危险性，现实线与副本线开始缠绕。

**现实案件类型来源**：火灾逃生/身份盗用（彻底虚构化）

**副本皮肤**：写字楼消防通道——应急灯闪烁，楼层指示牌错乱，门禁全部失效。

**关键冲突**：
- 公司内控风暴启动，主角被调查
- 证件/门禁开始失效
- 档案出现空白页
- 某份签字记录的主角签名被冒用

**证据推进**：
1. 主角发现自己的签名出现在某份审批文件上，但他没有签过
2. 门禁日志显示他在某个时段进入公司，但他当时在站内
3. 公司档案室发现空白页——某份调查报告被撕
4. HR系统显示他的合规分为85（下降了15分）

**反转**：
权限不是帮你，是把责任链压到你名下——你的签名被用来背书某套流程。

**恐怖点**：
- 消防通道的门全部锁死，应急灯变成红色
- 广播："身份验证失败，请到人事部门重新登记"
- 档案室的某个文件夹自动打开，里面是主角的档案

**行动线**：
- 主角被迫在现实中"取证"——证明自己没有签过那份文件
- 发现签名时间与站内时间重叠
- 意识到现实与站内存在"时间同步"

**奖励/代价**：
- 奖励：无
- 代价：身份缺页被发现，门禁失效

**章末悬念**：
周宁出现，点破"你在替别人完成出站"。

**台账更新**：F5（档案缺页）

## 第 4 章：封站（无菌站）

**目标**：副本皮肤切到更恐怖的场景，把"按流程会死人"具象化。

**现实案件类型来源**：尸体处理失误/失踪人口（彻底虚构化）

**副本皮肤**：殡仪馆冷藏间——温度计显示-18°C，铁柜编号混乱，地面有拖痕。

**关键冲突**：
- 按站务流程走会触发清场
- 破流程会触发惩罚
- 某份死亡证明的签字人失踪
- 冷藏柜里多出一具"无编号尸体"

**证据推进**：
1. 某份死亡证明缺少签字页
2. 签字人记录显示"已离职"，但离职时间在签字之后
3. 冷藏柜编号与日志不匹配——多出一具尸体
4. 地面拖痕指向封死的出口

**反转**：
处置建议能救人，但会牺牲一个人的身份完整度——你必须选择归责对象。

**恐怖点**：
- 冷藏柜的某个门自动打开，里面是空的
- 广播："检测到非合规操作，启动清场程序"
- 温度开始下降，应急灯变红

**行动线**：
- 主角在冷藏间内取证，时间紧迫
- 发现"无编号尸体"的死亡证明被篡改
- 选择归责"签字流程漏洞"，牺牲签字人的身份完整度

**奖励/代价**：
- 奖励：无
- 代价：目睹"按流程会死人"

**章末悬念**：
合规分阈值第一次被提及——低于60分会触发"身份回收程序"。

**台账更新**：无新伏笔

## 第 5 章：低分者

**现实案件类型来源**：非法拘禁/绑架（彻底虚构化）

**关键冲突**：
- 低分者在现实里失去身份证明
- 团队必须决定救不救
- 低分者的证件、银行卡、手机号全部失效
- 他"记得"的人都不记得他

**证据推进**：
1. 某个低分者的证件显示"无效"

**行动线**：

**奖励/代价**：

**章末悬念**：

## 第 11 章：证据的尸体

**目标**：证据成为可以杀人的东西，恐怖推到顶。

**现实案件类型来源**：证据污染/冤案制造（彻底虚构化）

**副本皮肤**：刑侦物证库——物证柜密集，某份证据袋缺号，地面有血迹。

**关键冲突**：
- 正确结论指向错误的人（被预设的可归责目标）
- 主角把"无法证明的真相"写进附录
- 某份证据袋的编号被涂改
- 物证库深处有"禁止打开"的柜子

**证据推进**：
1. 某份证据袋的编号被涂改，原始编号消失
2. 物证日志显示"已销毁"，但证据袋还在
3. 某份证明的归责对象是"预设目标"
4. 地面血迹指向某个物证柜

**反转**：
主角把"无法证明的真相"写进附录，触发未知后果——附录被系统"吞掉"。

**恐怖点**：
